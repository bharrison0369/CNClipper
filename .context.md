# CNClipper Context

CNClipper is a personal hobby/open-source project: a universal CNC module library for Klipper, providing reusable `klippy/extras/` Python modules for waterjet, plasma, laser, mill, EDM, lathe, and other CNC machine classes. Lives at `C:\GitHub\CNClipper`; GPL-3.0, targeting community adoption. Currently at Phase 0 (scope lock and feasibility) — no implementation code yet.

## Architecture & design principles

Modules are standalone `klippy/extras/` Python files, load only when referenced in `printer.cfg`, no core Klipper modifications by default. Cross-module dependencies via printer object lookups. Four-tier model (provisional): `core/` (primitives), `common/` (composed patterns), `machine/` (subset-shared), `machines/` (thin machine-specific). Python 3, GPL-3.0 matching Klipper. (source: .standards.md#Architecture)

## High-risk feasibility items

The following features are most likely to force core patches — prototype these first to bound fork risk:
1. True feed hold
2. Motion-synchronized I/O
3. General probing semantics
4. WCS beyond Klipper's current offset model
5. Universal `M3/M4/M5` interception and replacement
(source: docs/phase-0-scope-lock.md#High-Risk-Feasibility-Items)

## Baseline shared capability candidates

Strong candidates for shared scope: WCS/offsets, probing primitives, synchronized and immediate I/O, input wait semantics, universal tool output, coolant/process output, feed hold, feed/tool overrides, tool table/tool length, path control modes. (source: docs/phase-0-scope-lock.md#Initial-Baseline-Candidates)
