### [2026-04-11 14:30] CLAUDE [DESIGN]

- Updated module tier descriptions in STANDARDS.md to reflect refined architecture (core = primitives, common = composed patterns).
- Refined development plan in .agent-context.md with 4-phase structure grounded in CNC standards research:
  - Phase 1: Core primitives (cnc_wcs, cnc_io, cnc_tool_output, cnc_dwell)
  - Phase 2: Composed modules (cnc_output_sequencer, cnc_interlock, cnc_feed_hold, cnc_overrides)
  - Phase 3: Wazer waterjet hardware validation
  - Phase 4: Extended features (probing, tool table, coolant, path blending, canned cycles)
- Added research findings from CNC standards (RS274/NGC, grblHAL, LinuxCNC) and existing Klipper CNC projects (klipper-plasma, KCNC, klipper-for-cnc, klipper4cnc, grblHAL) to .agent-context.md
- Refined blocker analysis: feed hold and motion-synchronized I/O identified as the two hardest problems requiring early prototyping
- Files: STANDARDS.md, .agent-context.md, .agent-log/changelog.md
- Notes: Key design discussion with user - abstractions must be universal to attract community adoption beyond Wazer. A "pierce module" that only works for waterjet is wrong; it should be timed output sequencing that any machine configures. The 3-machine validation rule in STANDARDS.md enforces this.
